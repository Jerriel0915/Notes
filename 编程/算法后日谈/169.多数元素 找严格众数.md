给定一个大小为 `n` 的数组 `nums` ，返回其中的多数元素。多数元素是指在数组中出现次数 **大于** `⌊ n/2 ⌋` 的元素。

你可以假设数组是非空的，并且给定的数组总是存在多数元素。

**示例 1：**
	**输入：** $nums = [3,2,3]$
	**输出：** $3$

**示例 2：**
	**输入：** $nums = [2,2,1,1,1,2,2]$
	**输出：** 2

**提示：**
- `n == nums.length
- `1 <= n <= 5 * 104`
- `-109 <= nums[i] <= 109`
- 输入保证数组中一定有一个多数元素。

**进阶：** 尝试设计时间复杂度为 $O(n)$、空间复杂度为 $O(1)$ 的算法解决此问题。

---
## 朴素想法
看到计数脑海中第一个想到的就是哈希表了，额外开一个O(N)空间用于记录数字的出现次数，当次数符合要求时直接返回：
``` C++
class Solution {
public:
    int majorityElement(vector<int>& nums) {
        unordered_map<int, int> count;
        for (int num : nums) {
            if (++count[num] > (nums.size() - 1) / 2) {
                return num;
            }
        }
        return 0;
    }
};
```
- **时间复杂度：** $O(n)$ 。
- **空间复杂度：** $O(n)$。

---
## 进阶思路 1：排序
由题意很容易得出，严格众数与数组的元素顺序**无关**，那么不妨将原数组排个序。由于众数在整个数组中的占比大于 $⌊2n​⌋$ ，因此由众数组成的子列一定会跨越数组的中点（想象两根绳子，其中一根的长度比另一根的一半长，将两根绳子平行摆放），所以直接返回数组中点的元素就是所需的严格众数。
```C++
class Solution {
public:
    int majorityElement(vector<int>& nums) {
        sort(nums.begin(), nums.end());
        return nums[nums.size() / 2];
    }
};
```
- **时间复杂度：**  $O(nlog_2(n))$  ，排序算法所需时间复杂度为 $O(nlog_2(n))$。
- **空间复杂度：** $O(log_2(n))$ ，使用语言自带的排序算法需要使用 $O(log_2(n))$ 的栈空间。

---
## 进阶思路 2：Boyer-Moore 投票算法
把众数记为 $+1$，把其他数记为 $−1$，将它们全部加起来，显然和大于 $0$，从结果本身我们可以看出众数比其他数多。
具体来说，不妨假设 `nums[0]` 为候选众数，维护一个全局的分数 `round = 0` ，依次遍历数组，执行上述的计分制。
	当 `round == 0` 时，让此时的 `nums[i]` 成为新的候选众数。
	遍历结束后，候选众数即为所求的严格众数。
```C++
class Solution {
public:
    int majorityElement(vector<int>& nums) {
        int stay = nums[0], round = 0;
        for (int num : nums) {
            if (num == stay) {
                round++;
            } else {
                round--;
            }

            if (round == 0) {
                stay = num;
                round++;
            }
        }
        return stay;
    }
};
```
- **时间复杂度：** $O(n)$。
- **空间复杂度：** $O(1)$。
